Here are three common ways to target and style elements.


1. **ID Selector (`#`)**  
Targets a single element with a unique ID. Use `#idName` in CSS to style the element with `id="idName"`.  
Example:  
```css
#header {
  background-color: blue;
}
```
```html
This is the header
```

2. **Class Selector (`.`)**  
Targets one or more elements with a shared class. Use `.className` in CSS to style elements with `class="className"`.  
Example:  
```css
.highlight {
  color: red;
}
```
```html
This text is red.
This text is also red.
```

3. **Element Selector (`div`)**  
Targets all elements of a specific type. Use the element name directly to style all such elements.  
Example:  
```css
div {
  padding: 10px;
  border: 1px solid black;
}
```
```html
All div elements get padding and border.
Another div.
```

***

### Summary:

| Selector       | Syntax   | Targets                          | Example         |
|----------------|----------|---------------------------------|-----------------|
| ID selector    | `#id`    | Single element by unique ID      | `#header`       |
| Class selector | `.class` | One or many elements by class    | `.highlight`    |
| Element selector | `element` | All elements of that type (e.g., `div`, `p`) | `div`          |

Each method helps you control styling at different levels of specificity and reusability.

Size in CSS

In CSS, size usually refers to controlling the width and height of elements. Here’s a quick overview.

Common Value Types

units: px (pixels), em, rem, cm etc.
Example: width: 500px;

Percentage: relative to the size of the containing element.
Example: width: 50%;

auto: lets the browser calculate the size (default).

Additional Related Properties max-width and max-height: sets maximum size limits. min-width and min-height: sets minimum size limits.

Min-Max Width in CSS

In CSS, min-width and max-width are properties used to control the minimum and maximum width of an element. They are very helpful for making responsive designs.

min-width

Specifies the minimum width of an element.;

The element’s width will never be smaller than this value, even if its content or container is smaller.;

Useful for preventing elements from becoming too narrow.

Additional Related Properties max-width and max-height: sets maximum size limits. min-width and min-height: sets minimum size limits.

Display Properties in CSS

The display property in CSS is fundamental for determining how HTML elements are rendered and interact in the layout flow of a webpage. It controls whether elements act as blocks, inlines, containers, or are hidden entirely. Here’s a concise overview:

What is the display Property?

The display property specifies the display behavior (the type of rendering box) of an element. Every HTML element has a default display value, but you can override it using CSS to control layout and structure.

Introduction

The display property in CSS is one of the most essential concepts for shaping how elements behave and interact within a web page layout. It dictates whether an element appears as a block, inline, grid, flex container, or is even visible at all. Understanding the display property is the foundation of building layouts in frontend development.

Block Display

When an element is given a block display type, it begins on a new line and stretches out as far as it can horizontally by default. Block-level elements allow you to specify width and height directly. Headings, paragraphs, and divs are common examples of default block elements. This layout is suited for major structural divisions within a site.

Example: A navigation bar stretches fully across the page’s width and appears on its own line.

Inline Display

Elements with an inline display remain within the flow of surrounding text, only taking up as much space as their content requires. Unlike block elements, inline elements do not begin on a new line and you cannot set their width or height. Links, spans, and emphasized text are common examples of inline elements.

Example: A piece of highlighted text appears within a paragraph without breaking the line.

Inline-Block Display

The inline-block display combines the characteristics of both inline and block elements. Like inline, the elements do not start on a new line, but like block, you can set their width and height. This is especially useful for things like buttons or smaller containers that need specific sizing but still flow inline with text or other elements.

Example: Several buttons are aligned horizontally that have fixed widths and heights but remain on the same line.

None (Hidden Elements)

Setting an element’s display property to none completely removes it from the page — it will not be visible and will not occupy any space. This is often used for hiding menus or modals, or for implementing features that appear based on user interaction.

Example: A hidden popup window or menu that is not visible and does not affect the surrounding layout.

Flex (Flexible Layouts)

The flex value turns an element into a flex container and its children into flexible items. Flex layouts provide a robust and simple way to create complex, responsive designs, such as horizontal navigation bars, or to center items both vertically and horizontally.

Example: A horizontal menu bar where the menu items automatically space themselves evenly across the container.

Grid (Grid Layouts)

Setting an element’s display to grid creates a grid container for its children, unlocking two-dimensional layout capabilities. Grids allow the precise arrangement of elements in rows and columns and are widely used for dashboard layouts or image galleries.

Example: A photo gallery where images are arranged in neat rows and columns with equal gaps between them.

Contents

With the contents display value, the container element itself becomes invisible in the layout flow, but its children remain as if the parent does not exist. This allows for cleaning up markup while retaining all child elements in the visual structure.

Example: Removing a wrapping element from a list so its items behave as direct children of the outer container.

List-Item

Elements given the list-item value behave as if they are part of a list, showing bullets or numbers where appropriate. Although <li> elements are typically list items, any element given this display type will be treated similarly, which is useful for generating custom lists with non-typical tags.

Example: Making individual blocks behave like items in a bulleted list, even if they are not inside a list container.

Table, Table-Row, Table-Cell

CSS allows non-table elements to behave like tables, rows, and cells. The table value causes an element to act like a table; table-row acts like a row within a table; and table-cell behaves like an individual cell. These are valuable for complex tabular data or custom layout structures without using actual HTML tables.

Example: Structuring content in rows and columns using div-like elements that behave exactly like a table with borders and alignment.

Inline-Flex and Inline-Grid

inline-flex and inline-grid are similar to flex and grid, but these containers behave as inline elements, remaining in the flow of text. This can be necessary when creating advanced layouts that must integrate seamlessly with inline text or other inline elements.

Example: A group of icons aligned neatly next to a paragraph without forcing a line break.

Run-in

The run-in value is rarely used, as its behavior depends on context. Sometimes elements display as blocks; other times as inline elements, based on their surrounding content.

Example: A heading that sometimes appears on the same line as the next block if space allows or starts on a new line otherwise.

Initial and Inherit

initial resets the display to its default value for that HTML element.
inherit causes the element to take the display value of its parent.

Example: Resetting a customized button’s layout to behave like a normal button (default) or making an element match its container’s display style.

Summary

In modern web development, the display property serves as the cornerstone for structuring page content. It determines if elements form the skeleton of the page (block), blend into lines of text (inline), form advanced responsive layouts (flex, grid), or even disappear entirely (none). Mastering its values and usage equips designers and developers with immense control and flexibility when building for the web.

Chapter: CSS Flexbox

Introduction

CSS Flexbox, or Flexible Box Layout, is a powerful layout module that allows you to design complex and responsive layouts more efficiently. It provides a method for aligning and distributing space among items in a container, even when their size is unknown or dynamic. Flexbox adapts the layout to different screen sizes and display devices, making it essential for modern web design.

Flex Container

The flex container is the parent element where flexbox is applied by setting display: flex; or display: inline-flex;. This container activates flexbox behavior on its child elements, called flex items.

Example:

<div style="display: flex;">
  <div>Item 1</div>
  <div>Item 2</div>
  <div>Item 3</div>
</div>

Flex Items

Flex items are the direct children of a flex container. They are laid out according to the rules of flexbox, which can control their size, alignment, order, and spacing.

Example: List items in a menu that are evenly spaced and can adjust size responsively. 

<nav style="display: flex; justify-content: space-around;">
  <a href="#">Home</a>
  <a href="#">About</a>
  <a href="#">Contact</a>
</nav>

Flex Direction

The flex-direction property controls the main axis direction along which flex items are laid out.

  • row: Items are laid out horizontally from left to right (default).
  • row-reverse: Items laid out horizontally but in reverse order.
  • column: Items laid out vertically from top to bottom.
  • column-reverse: Items laid out vertically but bottom to top.

Example: A sidebar menu using flex-direction: column; to stack items vertically.

<div style="display: flex; flex-direction: column;">
  <button>Dashboard</button>
  <button>Settings</button>
  <button>Profile</button>
</div>

Justify Content

This property manages how flex items are aligned along the main axis.

  • flex-start: Items start from the beginning of the container.
  • flex-end: Items align to the end.
  • center: Items are centered.
  • space-between: Items are evenly distributed with the first item at the start and last item at the end.
  • space-around: Items have equal space around them.

Example: Navigation links distributed evenly across a horizontal bar using justify-content: space-between;. 

<nav style="display: flex; justify-content: space-between;">
  <a href="#">Home</a>
  <a href="#">Services</a>
  <a href="#">About</a>
  <a href="#">Contact</a>
</nav>

Align Items

This controls alignment of flex items along the cross axis (perpendicular to the main axis).

  • stretch (default): Items stretch to fill the container.
  • flex-start: Items aligned to the start of the cross axis.
  • flex-end: Items aligned to the end of the cross axis.
  • center: Items centered along the cross axis.

Example: Centering menu icons vertically inside a navigation bar.

<nav style="display: flex; align-items: center; height: 60px;">
  <div>Logo</div>
  <a href="#">Home</a>
  <a href="#">About</a>
</nav>

Flex Wrap

By default, flex items try to fit on one line. Use flex-wrap to allow items to wrap onto multiple lines.

  • nowrap: No wrapping (default).
  • wrap: Items wrap onto multiple lines.
  • wrap-reverse: Items wrap but in reverse order.

Example: Product cards wrapping to new lines on smaller screens with flex-wrap: wrap;. 

<div style="display: flex; flex-wrap: wrap;">
  <div style="width: 200px; margin: 10px;">Card 1</div>
  <div style="width: 200px; margin: 10px;">Card 2</div>
  <div style="width: 200px; margin: 10px;">Card 3</div>
  <div style="width: 200px; margin: 10px;">Card 4</div>
</div>

Order

The order property controls the order in which flex items appear, regardless of their source order in HTML.

Example: Moving a “Sign Up” button to appear last even if it appears first in the markup.

<div style="display: flex;">
  <button style="order: 2;">Login</button>
  <button style="order: 1;">Sign Up</button>
</div>

Flex Grow, Flex Shrink, Flex Basis

  • flex-grow: Defines how much a flex item will grow relative to others.
  • flex-shrink: Defines how much a flex item will shrink relative to others.
  • flex-basis: Defines the initial size of a flex item before growing or shrinking.

Example: Allowing a search input box in a toolbar to grow and fill remaining space with flex-grow: 1;. 

<div style="display: flex;">
  <input type="text" style="flex-grow: 1;" placeholder="Search...">
  <button>Go</button>
</div>

Align Self

This property overrides the container’s align-items for individual flex items.

Example: Keeping one menu item aligned differently, like bottom aligned, in an otherwise centered menu.

<nav style="display: flex; align-items: center; height: 60px;">
  <div>Logo</div>
  <a href="#" style="align-self: flex-end;">Special Link</a>
</nav>

Summary

CSS Flexbox revolutionized layout design by offering flexible, dynamic, and responsive control over alignment, sizing, space distribution, and order of elements within a container. It's a cornerstone technique for modern web layouts and allows developers to build adaptable designs with less code and complexity.

Chapter: CSS Grid

Introduction

CSS Grid Layout is a powerful two-dimensional system that allows web developers to create complex and responsive grid-based layouts easily. Unlike Flexbox, which is primarily one-dimensional (either row or column), CSS Grid handles both rows and columns simultaneously, providing precise control over placement, sizing, and alignment of content in a grid format.

Grid Container

The grid container is the parent element on which you apply display: grid; or display: inline-grid; to enable grid layout for its children.

Example:

<div style="display: grid; grid-template-columns: 100px 200px 100px;">
  <div>Item 1</div>
  <div>Item 2</div>
  <div>Item 3</div>
</div>

Grid Template Columns and Rows

These properties define the number and size of columns and rows.

  • grid-template-columns: Defines the width of each column.
  • grid-template-rows: Defines the height of each row.

Example: Creating a grid with three columns and two rows:

<div style="
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
  grid-template-rows: 100px 200px;
">
  <div>Item 1</div>
  <div>Item 2</div>
  <div>Item 3</div>
  <div>Item 4</div>
  <div>Item 5</div>
  <div>Item 6</div>
</div>

Grid Gap

The gap property (or grid-gap) adds spacing between grid tracks, both rows and columns.

Example: Adding a 15px gap between grid items:

<div style="
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 15px;
">
  ... grid items ...
</div>

Grid Item Placement

Grid items can be placed precisely using line numbers or named grid areas.

  • grid-column-start and grid-column-end: Controls the starting and ending column lines for an item.
  • grid-row-start and grid-row-end: Controls the starting and ending row lines.

Example: An item spanning from column 1 to 3 and rows 1 to 2:

<div style="
  grid-column: 1 / 3;
  grid-row: 1 / 2;
">Spanning item</div>

Auto Placement

If you don’t specify grid lines for items, the grid automatically places them row-wise or column-wise depending on the configuration.

Responsive Grids

Using fractional units (fr), minmax(), and auto-fill features makes grids flexible and responsive.

Example: A responsive grid with columns that shrink but don’t get smaller than 150px: 

<div style="
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(150px, 1fr));
  gap: 10px;
">
  ... grid items ...
</div>

Summary

CSS Grid Layout offers developers precise and powerful tools to create two-dimensional layouts on the web. With easy control over rows, columns, gaps, and item placement, Grid complements other CSS layout methods are essential for building responsive, visually rich web designs.

Chapter: CSS Pseudo-elements

Introduction

CSS pseudo-elements allow you to style specific parts of an element’s content without needing to add extra HTML markup. They are useful for adding decorative content, styling the first letter or line of text, inserting content before or after an element, and more. Pseudo-elements are powerful tools for enhancing design and UX while keeping HTML clean.

Common Pseudo-elements

  • ::before - Inserts content before an element’s actual content.
  • ::after - Inserts content after an element’s actual content.
  • ::first-letter - Styles the first letter of an element.
  • ::first-line - Styles the first line of an element.
  • ::selection - Styles the portion of an element that’s selected by the user.

How to Use

Pseudo-elements are written with two colons :: followed by the pseudo-element name after a selector.

Examples

::before and ::after:

p::before {
  content: "Note: ";
  font-weight: bold;
  color: red;
}

p::after {
  content: " ✔";
  color: green;
}

This inserts the word “Note:” before every paragraph and a checkmark after it.

::first-letter:

p::first-letter {
  font-size: 2rem;
  font-weight: bold;
  color: blue;
}

This styles the very first letter of every paragraph to stand out.

::first-line:

p::first-line {
  font-variant: small-caps;
  color: gray;
}

This styles the first line of each paragraph.

::selection:

::selection {
  background-color: yellow;
  color: black;
}

This changes the background and text color of any text the user selects on the page.

Notes

  • Pseudo-elements do not create real DOM nodes—they are purely visual effects applied by CSS.
  • Always use double colon syntax :: for CSS3 pseudo-elements. Single colon : is still supported for backward compatibility with CSS2.

Summary

Pseudo-elements extend the power of CSS by letting you style specific parts of an element’s content in creative ways without extra HTML. They are key for advanced design effects, user interaction feedback, and cleaner code structure.

Chapter: CSS Pseudo-classes

Introduction

CSS pseudo-classes allow you to define a special state of an element without adding classes or IDs in the HTML. They enable styling based on user interaction (like hovering or focusing), element position, form state, or structural conditions within the DOM. Pseudo-classes enhance user experience and enable dynamic styling purely with CSS.

Common Pseudo-classes

  • :hover — Styles an element when the user hovers over it with a mouse.
  • :focus — Styles an element that has keyboard or mouse focus.
  • :active — Styles an element at the moment it is being activated (e.g., clicking a button).
  • :visited — Styles links that the user has already visited.
  • :first-child — Styles an element that is the first child of its parent.
  • :last-child — Styles an element that is the last child of its parent.
  • :nth-child(n) — Styles elements based on their position among siblings.
  • :not(selector) — Styles elements that do NOT match the selector.
  • :checked — Styles checked checkboxes or radio buttons.

How to Use

Pseudo-classes are appended to a selector with a single colon : followed by the pseudo-class name.

Examples

:hover - Change button background on mouse hover:

button:hover {
  background-color: #007BFF;
  color: white;
}

Explore more button hover effects: Top CSS Hover Effects

:focus - Outline an input field when focused:

input:focus {
  border-color: #00BFFF;
  box-shadow: 0 0 5px #00BFFF;
}

:nth-child(n) - Style every even list item:

li:nth-child(even) {
  background-color: #f0f0f0;
}

:not(selector) - Style all buttons except those with class 'disabled':

button:not(.disabled) {
  cursor: pointer;
}

:checked - Style checked checkboxes:

input[type="checkbox"]:checked {
  accent-color: green;
}

Pseudo-classes provide a flexible way to style elements based on their state, position, or user interaction, enhancing interactivity and visual feedback in web designs without modifying the HTML structure.
